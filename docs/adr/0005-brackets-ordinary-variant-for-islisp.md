# Brackets option gains a third `Ordinary` variant, ahead of implementing it

The `Brackets` option (part of the `Options`/`Dialect`-preset design from ADR-0003) originally had two variants: `List` (Scheme's `[]` as an alternate list delimiter) and `Vector` (Emacs Lisp's `[]` as a data-vector literal). Investigating ISLisp (ISO/IEC 13816) as a prospective future dialect surfaced a third case: the ISLisp core language assigns `[`, `]`, `{`, `}` no special delimiter meaning at all — per the working draft's lexeme character set (§6), they are ordinary symbol-constituent characters, unlike Scheme, Emacs Lisp, or Clojure. We add `Brackets::Ordinary` (and the equivalent for braces) now, while `Brackets` is still an internal, unshipped design, rather than deferring until ISLisp support is actually built — extending an already-public, exhaustively-matched enum later would be a breaking change for consumers, while adding the variant now costs nothing.
